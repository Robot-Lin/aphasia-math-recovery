# 移动端适配执行清单

## 项目概述
为算术康复练习工具添加移动端适配，确保在 iPhone/Android 主流机型上体验流畅。

---

## 🎯 适配原则
1. **触摸优先**: 按钮尺寸 >= 44px，间距足够
2. **响应式布局**: 使用 flex/grid + media queries
3. **字体可读**: 最小 14px，行高充足
4. **性能优化**: 减少动画，优化渲染

---

## 📋 子项目清单

### P0 - 核心布局适配 (影响所有页面)

#### [ ] 1. 侧边栏移动端折叠
- **文件**: `index.html`
- **问题**: 220px 固定侧边栏在移动端占用过多空间
- **方案**:
  - 小屏幕 (< 768px) 隐藏侧边栏
  - 添加汉堡菜单按钮
  - 侧边栏以抽屉形式从左侧滑出
- **验收标准**:
  - [ ] 屏幕宽度 < 768px 时侧边栏自动隐藏
  - [ ] 显示汉堡菜单按钮 (左上角)
  - [ ] 点击汉堡菜单侧边栏从左侧滑出
  - [ ] 点击遮罩层或滑动可关闭侧边栏
  - [ ] 主内容区宽度自适应填满屏幕

#### [ ] 2. 主内容区响应式调整
- **文件**: `index.html`, 所有页面 JS
- **问题**: margin-left: 220px 在移动端不适用
- **方案**:
  - 移动端移除 margin-left
  - 添加 padding 防止内容贴边
  - 最大宽度限制适配小屏幕
- **验收标准**:
  - [ ] 移动端主内容区无左边距
  - [ ] 内容区有合适的左右 padding (16px)
  - [ ] 内容不超出屏幕宽度

### P1 - 页面级适配

#### [ ] 3. 首页 (home.js) 移动端适配
- **文件**: `src/pages/home.js`
- **问题**: 卡片网格、统计图表可能在小屏幕上显示不佳
- **方案**:
  - 统计卡片改为单列或双列布局
  - 快捷入口按钮增大触摸区域
  - 推荐区域字体和间距调整
- **验收标准**:
  - [ ] 统计卡片在移动端垂直堆叠或双列
  - [ ] 快捷入口按钮 >= 44px 高度
  - [ ] 文字不折行或合理折行

#### [ ] 4. 智能练习设置页 (practice-settings.js)
- **文件**: `src/pages/practice-settings.js`
- **问题**: 难度选择按钮、类型选择网格
- **方案**:
  - 难度按钮垂直排列
  - 题目数量滑块适配触摸
  - 开始练习按钮增大
- **验收标准**:
  - [ ] 所有按钮触摸目标 >= 44px
  - [ ] 滑块易于拖动
  - [ ] 页面可滚动，不遮挡内容

#### [ ] 5. 练习页 (practice-keypad.js & practice-choice.js)
- **文件**: `src/pages/practice-keypad.js`, `src/pages/practice-choice.js`
- **问题**: 数字键盘、选项按钮布局
- **方案**:
  - 数字键盘改为 3x4 网格，按钮增大
  - 选项按钮垂直排列或增大间距
  - 题目显示字体自适应
- **验收标准**:
  - [ ] 数字键盘按钮 >= 60px
  - [ ] 选项按钮间距足够，避免误触
  - [ ] 题目数字在不同屏幕尺寸下清晰可见

#### [ ] 6. 基础训练页 (basic-training.js)
- **文件**: `src/pages/basic-training.js`
- **问题**: 左右分栏布局、训练卡片
- **方案**:
  - 移动端改为单栏垂直布局
  - 控制面板移到底部或折叠
  - 训练卡片全宽显示
- **验收标准**:
  - [ ] 移动端单栏布局
  - [ ] 题目显示字号适配
  - [ ] 按钮易于触摸

#### [ ] 7. 结果页 (result.js)
- **文件**: `src/pages/result.js`
- **问题**: 统计图表、数据展示
- **方案**:
  - 图表改为垂直布局
  - 分数和统计信息重新排版
- **验收标准**:
  - [ ] 结果数据清晰可读
  - [ ] 图表在移动端正常显示

#### [ ] 8. 错题本 (mistakes.js) & 历史记录 (history.js)
- **文件**: `src/pages/mistakes.js`, `src/pages/history.js`
- **问题**: 列表项、筛选按钮
- **方案**:
  - 列表项增大高度
  - 筛选器改为下拉或折叠
- **验收标准**:
  - [ ] 列表项触摸目标足够大
  - [ ] 筛选器易于操作

#### [ ] 9. 设置页 (settings.js)
- **文件**: `src/pages/settings.js`
- **问题**: 开关、滑块、按钮
- **方案**:
  - 设置项垂直排列
  - 开关按钮增大
- **验收标准**:
  - [ ] 所有开关易于触摸
  - [ ] 页面可滚动

### P2 - 细节优化

#### [ ] 10. 触摸目标优化
- **文件**: 所有页面
- **内容**: 确保所有按钮、链接、可点击元素 >= 44px
- **验收标准**:
  - [ ] 使用工具检查所有点击区域 >= 44x44px
  - [ ] 相邻按钮间距 >= 8px

#### [ ] 11. 字体和排版优化
- **文件**: 所有页面
- **内容**: 响应式字体大小、行高
- **验收标准**:
  - [ ] 正文最小 14px
  - [ ] 标题随屏幕尺寸缩放
  - [ ] 行高 >= 1.5

#### [ ] 12. 横屏/竖屏适配
- **文件**: 所有页面
- **内容**: 处理横屏布局
- **验收标准**:
  - [ ] 横屏时布局合理不拥挤
  - [ ] 关键功能在横屏下可用

#### [ ] 13. 输入法弹出处理
- **文件**: `practice-keypad.js`
- **内容**: 处理软键盘弹出时的布局
- **验收标准**:
  - [ ] 输入框不被键盘遮挡
  - [ ] 页面可滚动到输入区域

### P3 - 测试验证

#### [ ] 14. 真机测试
- **设备**: iPhone (Safari), Android (Chrome)
- **测试内容**:
  - [ ] iPhone 12/13/14 竖屏测试
  - [ ] iPhone 12/13/14 横屏测试
  - [ ] Android 主流机型测试
  - [ ] 触摸响应测试
  - [ ] 页面滚动流畅度

#### [ ] 15. 性能优化
- **内容**: 移动端性能检查
- **验收标准**:
  - [ ] Lighthouse 移动端评分 > 80
  - [ ] 首屏加载 < 3s
  - [ ] 动画流畅 60fps
